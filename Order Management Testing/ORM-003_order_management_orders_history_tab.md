# ORM-003: Order Management – Order History Tab

> **Summary:** Verify that the **Order History** tab displays completed orders correctly with a clean and properly aligned table layout.

**Preconditions:**  
- Vendor is logged in.  
- Vendor is at Orders page.

| # | Step | Expected Behavior |
|---|------|-------------------|
| 1 | Click the **Order History** tab. | Tab loads successfully. |
| 2 | Observe table layout. | Columns (Customer, Order ID, Items, Date, Location, Fulfillment, Status, Total, Actions) are visible, aligned, and readable. |
| 3 | Check row formatting and color contrast. | Rows are evenly spaced with clear text and consistent styling. |
| 4 | Verify order details. | Data matches backend records (customer name, ID, date, location, fulfillment, status, total). |
| 5 | Check responsiveness. | Table adjusts correctly to different screen sizes without misalignment. |

**Actual Result:**  
System displays completed orders accurately in a clean, well‑formatted table.

Pass ✅
